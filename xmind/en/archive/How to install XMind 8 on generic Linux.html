How to install XMind 8 on generic Linux

Prepare files

 Download from http://www.xmind.net/download/linux/

  Ubuntu

   Click "Download XMind for Linux"

   Click "Save File" and hit "OK"

Extract

 Change directory to ~/Downloads

  ~$ cd ~/Downloads ~/Downloads$ ls xmind-8-update7-linux.zip ~/Downloads$

 Extract the file to directory xmind with the -d option

  ~/Downloads/xmind$$ unzip xmind-8-linux.zip -d xmind

Move to /opt/

 ~/Downloads$sudo mv xmind /opt

 Error

  "your account" is not in the sudoers file. This incident will be reported

  "mv: cannot move 'xmind' to '/opt/xmind' : Directory not empty

   ~/Downloads$sudo mv xmind /opt/

   gives the "Directory not empty" error. Use xmind/* instead of xmind

Prepare directories

 Allow writes to

  configuration directory

   $ sudo chmod a+w /opt/xmind/XMind_amd64/configuration

  data directory

   $ sudo mkdir -p /opt/xmind/workspace

   $ sudo chmod a+w /opt/xmind/workspace

Edit configuration file /opt/xmind/XMind_amd64/XMind.ini

 $ sudo nano /opt/xmind/XMind_amd64/XMind.ini

 Old

  -configuration ./configuration -data ../workspace -startup ../plugins/org.eclipse.equinox.launcher_1.3.200.v20160318-1642.jar --launcher.library ../plugins/org.eclipse.equinox.launcher.gtk.linux.x86_64_1.1.400.v20160518-1444 --launcher.defaultAction openFile --launcher.GTK_version 2 -eclipse.keyring @user.home/.xmind/secure_storage_linux -vmargs -Dfile.encoding=UTF-8

 New

  -configuration /opt/xmind/Xmind_amd64/configuration -data /opt/xmind/workspace -startup /opt/xmind/plugins/org.eclipse.equinox.launcher_1.3.200.v20160318-1642.jar --launcher.library /opt/xmind/plugins/org.eclipse.equinox.launcher.gtk.linux.x86_64_1.1.400.v20160 --launcher.defaultAction openFile --launcher.GTK_version 2 -eclipse.keyring @user.home/.xmind/secure_storage_linux -vmargs -Dfile.encoding=UTF-8

   Changes

    Line2 change: ./configuration

     /opt/xmind/Xmind_amd64/configuration

    Line 4 change: ../workspace

     /opt/xmind/workspace

     This works, too. But I prefer not to make ~/workspace because the directory structure gets messy.

      /home/USERNAME/workspace

       e.g.

        /home/mrtonnet/workspace

        /home/the.kim/workspace

    Line 6 change: ../plugins/*.jar

     /opt/xmind/plugins

      e.g.

       line6: /opt/xmind/plugins/org.eclipse.equinox.launcher_1.3.200.v20160318-1642.jar

    Line 8 change: ../plugins/*.v20160518-1444

     line8: /opt/xmind/plugins/org.eclipse.equinox.launcher.gtk.linux.x86_64_1.1.400.v20160518-1444

    If this doesn't work for line 6 & 8, check the file names.

     $ls /opt/xmind/plugins/org.eclipse.equinox.launcher_*

     $ls /opt/xmind/plugins/org.eclipse.equinox.launcher.gtk*

Prepare fonts

 Create fonts directory

  $ sudo mkdir -p /usr/share/fonts/truetype/xmind

 Copy fonts

  $ sudo cp -R /opt/xmind/fonts/* /usr/share/fonts/truetype/xmind/

 Refresh font cache

  $ sudo fc-cache -f

  It may take a while. Please wait. It took about 30 seconds to me.

Create menu item

 Create file /usr/share/applications/xmind.desktop

  [Desktop Entry] Name=Xmind8 Pro Exec=/opt/xmind/XMind_amd64/XMind %F Icon=/opt/xmind/xmind_icon.png GenericName=MindMap Editor Comment=Record and edit MindMap files Type=Application Categories=Utility;Editor;Application;Office StartupNotify=false Terminal=false

 Refresh gnome-shell if needed

  Press [Alt]+[F]

  type r

  Press [Enter]

Download the icon.

 Download the icon file

  e.g.

   

  at

   http://s3.amazonaws.com/xmindnet/blog/en/new_branding_new_logo

   http://www.xmind.net/blog/en/2013/11/

  If the link is broken, google-search an Xmind icon file with "xmind icon" and download a .png or .xpm file.

 Copy the downloaded file to /opt/xmind

  sudo mv ~/Download/new_branding_new_logo.png /opt/xmind

 Rename the file name to xmind_icon.png

  sudo mv /opt/xmind/new_branding_new_logo.png /opt/xmind/xmind_icon.png

 Note

  If you want a different file name, it's fine as long as the new file name is identical to Icon=/opt/xmind/your_new_icon_file_name
